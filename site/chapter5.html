5.1 Information Asset Security Policies, Frameworks, Standards, and Guidelines

Overview

Security policies are the foundation of any information security program. They define acceptable behavior, controls, and governance.

Policy Elements (8 Core)

Element Description
Scope Who and what the policy covers
Policy Statement High-level intent and rules
Objectives CIA (Confidentiality, Integrity, Availability)
Guidance Specific instructions
Monitoring How compliance is measured
Governance Oversight responsibilities
Compliance Legal/regulatory requirements
Privacy Data protection requirements

Characteristics of a Good Security Policy

  • Executive support (management commitment)
  • Business-driven (aligned with business goals)
  • Simple and accessible
  • Integrated with other policies
  • Strategic but with clear guidance
  • Specifies penalties for violations
  • Flexible and regularly updated

Types of Security Policies

Type Description Examples
Organizational/Master Enterprise-wide security Overall IS security policy
System-Specific (SysSP) For specific systems/technologies Firewall policy, server policy
Issue-Specific (ISSP) For specific security issues Acceptable use, BYOD, remote access, incident response, change control, IAM, access control

🎯 EXAM TIP: ISSP = Issue-Specific Security Policy. Examples include AUP, BYOD, remote access, change control, and incident response policies.

Standards vs. Frameworks

| Standards | Frameworks |--|-----------|------------| Nature | Mandatory rules | Processes/guidelines defining policies | Flexibility | Less flexible | More flexible | Purpose | Compliance baseline | Strategic guidance |

Framework Description Key Feature

  • Purpose: Ensure all systems meet a standard starting point
  • Topics covered:
  • Environment/inventory management
  • Malware protection
  • Password policies

  • Minimizing services (attack surface reduction)
  • Vulnerability management and auditing

5.2 Physical and Environmental Controls

Overview

Physical controls protect information assets from physical threats like unauthorized access, environmental hazards, and equipment damage.

Power Failure Types and Controls

Type Description Duration Solution
Blackout Total power loss Any duration UPS + Generator
Brownout Reduced voltage (108-125V AC) Extended Voltage regulator, UPS
Sag/Spike Brief voltage drop/surge Milliseconds Surge protector
Surge Extended overvoltage Seconds UPS with surge protection
EMI Electromagnetic interference — Shielding
Duration Solution
Very short (ms) Surge protectors
Short-intermediate (seconds – 30 min) UPS (Uninterruptible Power Supply)
Long-term (hours-days) Diesel/gasoline/propane generators

Fire Detection and Suppression

  • Water detectors: Under raised floors
  • Smoke detectors: Above AND below ceiling tiles; linked to monitored station with audible alarms
  • Fire suppression systems: Auto-activate on high heat
  • Wet pipe sprinklers: Water always in pipes; fastest response
  • Dry-pipe: Water held back by pressurized air (used in cold environments)
  • FM-200 (HFC-227ea / heptafluoropropane): Clean agent, safe for electronics
  • Novec 1230: Green alternative clean agent
  • Types of application: Total flooding vs. local application

Computer Room Placement Best Practices

  • Best location: Middle floors (3rd–6th)
  • Avoid: Basement (flooding), top floor (roof leaks, fire), external walls
  • Avoid proximity to: Water/gas pipes, paper storage, airports, chemical plants, railways

Physical Access Controls

Control Description
Bolting locks Metal key; should have "do not duplicate" marking
Combination/cipher locks Change combinations when employee leaves
Electronic door locks Card/token/biometric; individually assignable; can be deactivated remotely
Biometric locks Fingerprint, retina, etc.
Mantrap/Airlock Two interlocking doors; one person at a time; prevents piggybacking
Deadman doors Special mantrap for delivery areas
ID badges Different colors for visitors vs. employees
Video cameras Motion-activated; strategically placed; recorded
Security guards Must be bonded if external
Lighting Illuminates entry points

🎯 EXAM TIP: Mantrap/airlock = prevents piggybacking/tailgating. Piggybacking is a physical access exposure.

Physical Access Exposures

  • Unauthorized entry
  • Vandalism and theft
  • Copying, alteration, disclosure of data
  • Blackmail/embezzlement
  • Wiretapping

Perpetrators: Disgruntled employees, former employees, outsiders, accidental actors

Auditing Physical Security

  • Tour computer site including above ceiling tiles and below raised floors
  • Check all paths of entry (not just main doors)
  • Verify visitor logs and escort procedures

ICS/SCADA Security

Types: SCADA (Supervisory Control and Data Acquisition), PLCs (Programmable Logic Controllers), RTUs (Remote Terminal Units)

ICS Security Risks: - Poor security posture / default passwords - Improper input validation - Improper authentication - Poor configuration - Embedded legacy software - Network design weaknesses - Technology depreciation - Incompatibility with modern standards - Weak protocols - Dependencies on ICT infrastructure - Third-party presence - Compliance risk

ICS Best Practices: - Asset inventory - Network baseline monitoring - Network segmentation (firewalls understanding ICS protocols) - Principle of Least Privilege (POLP) - IDS/IPS deployment - Secure remote access - Physical security - Redundancy - System hardening - Incident Response Plan (IRP) - Supply chain security

5.3 Identity and Access Management (IAM)

Overview

IAM is the first line of defense in information security. It controls who can access what resources.

Key principle: Identification is separate from authentication.

IAM Components (4)

  1. Authentication Services — Verify identity claims
  2. Authorization Services — Determine what authenticated users can access
  3. User Management — Provisioning and deprovisioning of accounts
  4. Central User Repository — Centralized directory (e.g., Active Directory)

IAM Life Cycle

Enrollment → Role Determination → Provisioning → Review/Updating → Deprovisioning

🎯 EXAM TIP: Privilege creep occurs at the Review/Updating stage when accumulated access rights exceed current job needs.

Authentication Types

Type Factor Example
Password Something you know PIN, passphrase
Token Something you have Smart card, hardware token; used in REST APIs
Biometric Something you are Fingerprint, iris, retina
Certificate-based Something you have Digital certificate
MFA 2+ factors Password + OTP
Adaptive/Risk-based Context-based Location, device, behavior
LDAP Directory bind Active Directory (most common LDAP)
Smart card Possession + PIN PKI-based card

Single Sign-On (SSO)

  • Concept: One set of credentials for access to multiple systems
  • Architecture: Primary domain authenticates; secondary domains trust the primary
  • Advantages: Fewer passwords to manage, simplified administration, reduced overhead
  • Disadvantages: Single point of failure, OS support challenges, costly to implement

Authorization Types

Type Description Use Case
Coarse-grained High-level, role-based General access control
Fine-grained Detailed conditions, attributes Enforces POLP strictly

Access Control Models

Model Full Name Key Feature
MAC Mandatory Access Control Default deny; system enforced; not user-modifiable; for sensitive environments
DAC Discretionary Access Control Data owner controls access; additional filter over MAC
RuBAC Rule-Based Access Control ACL-defined rules; equal for all users
RBAC Role-Based Access Control Based on user roles; industry standard
ABAC Attribute-Based Access Control User + resource + environment attributes
PBAC Policy-Based Access Control Business roles + policies; used in ZTA

🎯 EXAM TIP: MAC = system enforced, cannot be changed by users. DAC = user/owner controlled. RBAC = most common in industry.

Zero-Trust Architecture (ZTA)

  • Principle: "Never Trust, Always Verify"
  • Approach: Session-based access; POLP; dynamic PBAC; continuous monitoring
  • Best practices: Monitor all traffic, update devices, POLP, segment network, use security keys for MFA, threat intelligence integration

Privileged Access Management (PAM)

Types of privileged accounts: - Administrator accounts - Emergency/"break-glass" accounts - Service accounts - Application accounts - SSH keys

Benefits of PAM: Reduced attack surface, POLP enforcement, streamlined rights, monitoring, secure access

Challenges: Credential management, tracking activity, ineffective threat analysis, cloud access, authentication vulnerabilities

Biometrics

Metric Full Name Description
FER Failure to Enroll Rate Cannot enroll in system
FRR / Type 1 False Rejection Rate Legitimate user rejected (false negative)
FAR / Type 2 False Acceptance Rate Impostor accepted (false positive)
CER/EER Crossover Error Rate / Equal Error Rate Where FRR = FAR; measures overall accuracy

🎯 EXAM TIP: Lower CER = better biometric accuracy. FAR = Type 2 error (more dangerous — lets bad people in).

Biometric accuracy ranking (best CER to worst): Palm > Hand geometry > Iris > Retina > Fingerprint > Voice

Federated Identity Management (FIM)

Protocol Type Key Use
SAML XML-based Authentication + authorization across domains
OAuth Token-based Authorization only; no credentials shared
OIDC JSON-based Adds identity layer to OAuth 2.0
SCIM Provisioning Cross-domain identity management; keeps info updated

| FIM | SSO |--|-----|-----| Architecture | Decentralized | Centralized | Scope | Cross-domain/cross-organization | Within same org/domain | Relationship | FIM can support SSO across domains | SSO is subset of FIM |

Password Security Best Practices

  • Minimum length per threat model (modern: 8+ chars, recommended 12+)
  • No dictionary words or personal details
  • History: last 13 passwords cannot be reused
  • Change: 30-day cycle for privileged; 90-day for regular users
  • Max 5 failed attempts → account lockout
  • 15-minute inactivity → automatic timeout
  • Always encrypted in storage AND transit

Password Attacks

Attack Description Defense
Sniffing Capture plaintext passwords Encryption in transit
Phishing Trick user into revealing password Security awareness
Brute force Try all combinations Account lockout, complexity
Dictionary Try common words Complexity requirements
Keylogger Record keystrokes Endpoint security
Rainbow table Precomputed hash lookup Salting
Credential stuffing Use leaked credentials on other sites MFA, monitoring
Password spraying Few passwords tried on many accounts Lockout policies

Identity Governance and Administration (IGA)

IA elements: Connectors, automated workflows, provisioning lifecycle, entitlement management IG elements: Separation of Duties (SoD), user access review, RBAC, analytics/reporting

IDaaS (Identity as a Service)

  • Cloud-based IAM subscription
  • Supports SSO, MFA, user identity management, provisioning, cloud directory
  • Benefits: Reduced costs, improved security, scalability, flexibility
  • Risks: Internet dependency, limited customization, security concerns, compliance, integration issues

5.6 Data Encryption

Overview

Encryption protects data confidentiality in transit and at rest, deters unauthorized alteration, and verifies authenticity.

Encryption Elements

Security = Algorithm + Key + Key Length
Longer key → Harder to brute force

Symmetric vs. Asymmetric Encryption

| Symmetric | Asymmetric |--|-----------|------------| Keys | Same key for encrypt/decrypt | Public key encrypts; private key decrypts | Distribution | Key shared out-of-band (challenge) | Public key freely distributed | Speed | Faster | Slower | Key length | Shorter | Longer | Digital signing | Cannot sign | Can sign (private key encrypts hash) | Use case | Bulk data encryption | Key exchange, digital signatures | Examples | DES (withdrawn), 3DES, AES, RC4 | RSA (1024-4096 bit) |

🎯 EXAM TIP: AES = current symmetric standard (128, 192, or 256-bit). RSA = primary asymmetric algorithm.

Hashing

  • Purpose: One-way function; integrity verification
  • Output: Fixed-length regardless of input size
  • No key required (optional salt)
  • Cannot be reversed
Algorithm Output Size Status
MD5 128 bits Weak; avoid
SHA-1 160 bits Deprecated
SHA-256 256 bits Current standard
SHA-512 512 bits High security

🎯 EXAM TIP: Industry is transitioning from SHA-1 to SHA-2/SHA-3. Salting defeats rainbow table attacks.

Encryption vs. Hashing Comparison

| Encryption | Hashing |--|------------|---------| Direction | Two-way (reversible) | One-way (irreversible) | Output length | Variable | Fixed | Key required | Yes | No (optional salt) | Purpose | Confidentiality | Integrity verification |

Encryption Modes

Mode Description Headers Encrypted? Security
Link encryption All traffic including headers; decrypts at each hop Yes Simple but complex key distribution
End-to-End (E2EE) Stays encrypted from source to destination No More secure

Advanced Encryption Types

ECC (Elliptic Curve Cryptography)

  • Smaller keys than RSA for equivalent security
  • Faster key generation
  • Modern RSA successor

Quantum Cryptography

  • Uses photons through fiber optic cables
  • QKD (Quantum Key Distribution) detects eavesdropping
  • Limitations: Polarization changes, max ~500km range, high cost, single-destination per channel

Homomorphic Encryption

  • Performs operations on encrypted data without decrypting
  • Types:
  • Partially: Infinite operations of one type (e.g., RSA is partially homomorphic)
  • Somewhat: Finite number of any operation
  • Fully: Infinite additions OR multiplications
  • Benefits: Supply chain security, cloud data security, compliance (GDPR), data analytics
  • Challenges: Inefficiency, high storage requirements, slow performance

Digital Signatures

  • Process: Sender signs with private key (hash → encrypt hash)
  • Verification: Recipient verifies with sender's public key
  • Provides: Integrity, Authentication, Nonrepudiation
  • Does NOT provide: Protection against unauthorized reading, theft, or copying
  • Replay attack protection: Add timestamp or counter

Digital Envelope

  • Message encrypted with symmetric key
  • Symmetric key encrypted with recipient's public key
  • Example: PGP (Pretty Good Privacy)
  • Used in S/MIME

TLS (Transport Layer Security)

  • Replaces SSL (SSL had vulnerability found in 2014)
  • Two protocols: 1. Handshake Protocol: Negotiates and establishes secure connection 2. Record Protocol: Actual data transmission; encryption + authentication
  • Phases: Peer negotiation → Key exchange/authentication → Symmetric encryption
  • HTTPS = HTTP + TLS

IPSec

Mode What's Encrypted Use Case
Transport mode Data portion (ESP payload) only Host-to-host
Tunnel mode Entire packet including header VPN, gateway-to-gateway
  • ESP (Encapsulating Security Payload): Provides confidentiality
  • AH (Authentication Header): Provides nonrepudiation
  • SA (Security Association): Defines security parameters; managed by IKE

Kerberos

  • Ticket-based SSO for distributed environments
  • Current version: Kerberos 5 (uses AES)
Component Purpose
KDC (Key Distribution Center) Trusted third party; provides authentication services
TGS (Ticket-Granting Server) Validates ticket for specific purpose
AS (Authentication Server) Hosts TGS; verifies authenticity of tickets
TGT (Ticket-Granting Ticket) User authentication token
ST (Service Ticket) Proof of authorization to access a specific resource

🎯 EXAM TIP: KDC is a single point of failure in Kerberos. If KDC offline → no authentication. Strict time requirements → clock sync critical.

SSH (Secure Shell)

  • Encrypted remote login; replaced Telnet
  • Layer: Application layer (not network layer like IPSec)
  • Best practices: Centralize key management, change default port, disable root login, implement key attribution, key rotation, continuous audits, implement firewall technology

DNSSEC

  • Adds digital signatures to DNS records
  • Prevents DNS cache poisoning and false zones
  • Risks introduced by DNSSEC: DDoS amplification, increased query response volume, slow performance (UDP-based)

Email Security

Technology Purpose
S/MIME Authentication + confidentiality using X.509 certificates
MOSS Multiple-party digital signatures; rarely used
PEM Authentication, integrity, confidentiality, nonrepudiation; uses RSA + DES
DKIM Detects forged sender addresses
PGP Open-source; authentication, integrity, privacy, nonrepudiation

Email Attacks: - Bombing: Sending identical emails repeatedly to one address - Spamming: Mass unsolicited emails; managed by SPF, Bayesian filtering, greylisting - Spoofing: Email appears from different source than actual sender - BEC (Business Email Compromise): Targets employees authorizing financial transactions

Email Controls: Spam filters, email encryption, antivirus, SEG (Secure Email Gateway), EAC (Email Attachment Control)

5.7 Public Key Infrastructure (PKI)

Overview

PKI is a system for managing public key certificates through a trusted hierarchy.

PKI Components: Policies, procedures, hardware, software, personnel for managing public key certificates

Digital Certificates

  • CA (Certificate Authority): Issues and manages certificates (e.g., Verisign, Thawte)
  • CP (Certificate Policy): Identifies actors, roles, duties, responsibilities in PKI
  • CPS (Certificate Practice Statement): How the CA issues certificates; includes certificate types, policies, algorithms used, key length, certificate lifetime

RA (Registration Authority)

  • Delegated administrative functions for a specific community
  • RA functions:
  • Verify subject information
  • Verify right to requested attributes
  • Proof of possession of private key
  • Report key compromise
  • Assign names
  • Initiate registration with CA
  • Initiate key recovery
  • Distribute physical tokens (smart cards)

Key Management Activities

Activity Description
Key creation Generating keys via key generator
Key distribution Secure transfer to user/system
Key storage/custody Secure storage; dual custody (2+ people required)
Key rotation Retiring old keys; implementing new ones
Key recovery/backup Key escrow for recovery
Key destruction Suspension, revocation, expiration, or destruction

Certificate Revocation Reasons

  1. Changes in affiliation (terminated, resigned, died)
  2. Private key compromise
  3. Cessation of operation (server decommissioned)
  4. Superseded (certificate replaced)
  5. Unspecified
  6. CA compromise

CRL (Certificate Revocation List)

  • Published periodically by CA
  • States: Revoked (irreversible) or Hold (reversible/temporary)
  • CRL should include: Issuer name, revocation date and time, reasons, specific time period, certificate extensions, signature algorithm, serial number, next CRL date

OCSP (Online Certificate Status Protocol)

  • Real-time certificate status check (valid/revoked/unknown)
  • OCSP Stapling: Server caches and "staples" OCSP response to its certificate → improves performance, reduces CA load, enhances privacy

PKI Risks

  • Outdated protocols
  • Weak cryptographic keys (<2048 bits)
  • Infrequent key rotation
  • Mismanaged certificates (expired certificates → outages)
  • Lack of automation
  • Insufficient skills and resources
  • Unclear certificate ownership
  • Lack of policies
  • Limited visibility (no centralized inventory)
  • Poor private key management
  • Compromised root CA → entire chain of trust broken (store root CA offline!)
  • Poor patch management

5.8 Cloud and Virtualized Environments

Virtualization vs. Cloud

| Virtualization | Cloud |--|---------------|-------| Definition | Multiple OS on one physical server | Delivers virtualized resources on demand over internet | Control | Organization controls | Shared responsibility | Location | On-premises | Off-premises |

Hypervisor Types

Type Description
Bare metal/Native Hypervisor runs directly on hardware; no host OS
Hosted Hypervisor runs on top of host OS
Containerization Containers share kernel with host OS

Virtualization Risks

  • Rootkits as hypervisors (below OS level; antivirus may not detect)
  • Hypervisor misconfiguration → unauthorized resource access
  • Guest tools attack vectors
  • Snapshots contain sensitive RAM data
  • Hosted virtualization lacks hypervisor access controls

Virtualization Controls

  • Strong physical/logical access over host and management console
  • Configuration management and system hardening
  • Network separation (avoid VMs in DMZ; management tools on separate network)
  • Strong change management

Virtual Circuits

Type Description Security
PVC (Permanent Virtual Circuit) Always available; like leased line Requires constant monitoring
SVC (Switched Virtual Circuit) On-demand; disassembled after use More secure; less monitoring needed

VLAN

  • Hardware-imposed network segmentation by switches
  • Static VLAN: Port-based assignment
  • Dynamic VLAN: Negotiated based on MAC/IP address
  • Benefits: Controls broadcast traffic, reduces sniffers, protects against broadcast storms, performance and security

🎯 EXAM TIP: VLANs ≠ Subnets. VLANs created by switches; subnets created by IP addressing.

VSAN (Virtual Storage Area Network)

  • Logical partition for VM storage; used in virtualized/cloud environments

| SAN | VSAN |--|-----|------| Infrastructure | Dedicated physical hardware | Uses existing host resources | Scalability | Difficult; requires new hardware | Dynamic allocation | Cost | Costly; specialized hardware | Less costly | Performance | Specialized hardware optimized | Caching and data mirroring |

SDN (Software-Defined Networking)

  • Separates control plane from data plane

Three planes: 1. Data plane: Forwards actual user data (TCP/IP) 2. Control plane: Dictates path flows; where admin manages the network 3. Management plane: Performance, fault management, configuration of remote devices (SNMP)

SDN Advantages: Centralized control, network abstraction, scalability, enhanced security, lower costs, network programmability

SDN Disadvantages: Latency, limited device management, network complexity, dependency on controller (single point of failure)

SDN Attacks: - Unauthorized access via compromised controller - Data loss (credentials stolen via switches) - Data modification (MITM if TLS not mandatory) - DoS on SDN controller (bogus calls → packet flood) - Malicious/compromised third-party applications

Containerization

  • Single OS with multiple isolated user spaces
  • Tools: Docker (building/sharing/running containers) + Kubernetes (orchestration at scale)

Benefits: Faster deployment, platform-agnostic, improved security (isolation), flexibility, operational efficiency, cost reduction, scalability, portability

Limitations: - Well-supported on Linux; limited on Windows - Shared kernel vulnerabilities (if kernel compromised, all containers affected) - Networking complexity - Monitoring challenges - Requires specialized expertise

Container Security Best Practices: - Systems hardening - Monitoring with specialized tools - Continuous auditing (logs critical) - Vulnerability assessments - Patching (update base image → rebuild application image) - Isolation (run containers from VM for additional isolation) - Incident response planning (forensics limited in containers) - Secure container management stack - Secure images

Virtualization vs. Containerization

| Virtualization | Containerization |--|---------------|-----------------| Isolation | Full OS isolation from host | Lightweight isolation; shared kernel | OS | Multiple complete OS | Single host OS | Guest support | Various OS (Windows, Linux) | Must match host OS | Deployment | Each VM with its own hypervisor | Multiple containers on single host | Storage | VHD per VM | Local disks per node | Load balancing | Failover clusters | Kubernetes orchestrator |

Cloud Migration (5.8.7)

Three forms: On-premises to cloud, cloud-to-cloud, reverse (cloud to on-premises)

Security Risks: - Multi-tenancy risks (unaware of other tenants) - API vulnerabilities - Compliance risk (data protection regulations) - Uncontrolled growth - Data loss during migration - Data security (insecure APIs, malware in transit) - Legacy architecture incompatibility - Reduced visibility (transferred to CSP) - Data remanence (incomplete data deletion) - Talent gap

Security Steps: - Develop secure migration plan - Assess current security posture - Establish security standards - Train employees on cloud security - Secure DevSecOps pipeline - Evaluate regulatory requirements - Encrypt data with TLS/HTTPS - Enable strict access control - Automate migration process - Data backup

Shared Responsibility Model (SRM)

  • CSP always responsible for: Physical security of cloud infrastructure
  • Customer always responsible for: Data security, GRC in the cloud
  • Shared: Security training, patch management, configuration management

Cloud Deployment Models

Model Description Security Level
Public CSP owned; accessible to everyone Least secure
Private Dedicated infrastructure; customer has more control High security
Hybrid Mix of public and private Medium-high security
Community Shared by member organizations High (like private)
Multi-cloud Multiple public clouds Can have loopholes; increased attack surface

Cloud Service Models

Model Description Target Users
IaaS Infrastructure (servers, storage, network) Sysadmins wanting full OS/app freedom
PaaS Application development sandbox Developers
SaaS Complete applications on demand End users

Key Cloud Risks (CSA Pandemic Eleven)

  1. Insufficient IAM and key management
  2. Insecure interfaces and APIs
  3. Misconfiguration and inadequate change control
  4. Lack of cloud security architecture and strategy
  5. Insecure software development
  6. Unsecured third-party resources
  7. System vulnerabilities
  8. Accidental cloud data disclosure
  9. Misconfigured container/serverless workloads
  10. Organized crime, hackers, APTs

Data Acquisition

  • Transfer all evidence to controlled location
  • Check all devices for write-protection (write-blocker device)
  • Capture volatile data (open ports, files, processes, user logons, RAM) before shutdown
  • Imaging: Bit-for-bit copy of entire disk (sector by sector); captures deleted files, fragments, residual data

  • Contents: Who had access (chronological), procedures followed, proof copies are identical to original (checksums, timestamps)

🎯 EXAM TIP: If you boot a suspected system before imaging, you write data to the hard drive — this can corrupt evidence and break chain of custody.

Best Practices for Digital Evidence

  • Prevent contamination: Never work on original data; use bit-for-bit images
  • Clear chain of custody: Audit logs; maintain complete record of access
  • Implement tamper detection: Cryptographic hash values
  • Secure storage devices: Password protection; off-site storage
  • Keep mobile devices digitally isolated: Faraday bag; airplane mode; avoid plastic bags (static)
  • Encrypt the hard drive: If heightened security concerns
  • Involve forensic experts: Know when to hand over to specialists
  • Avoid changing device power status: Leave on if on; off if off
  • Monitor evidence transactions: Periodic sign-out records
  • Install write-blocking software: Read-only access to preserve integrity
  • Incorporate other evidence: Internet activity, chat rooms, email headers

📝 PROBABLE EXAM QUESTIONS

Section 5.1 — Policies and Frameworks

  1. Q: Which framework provides IT governance and links IT to business goals? - A: COBIT

  2. Q: Which access control policy would be MOST appropriate for an employee who is switching departments? - A: RBAC — Role-based access ensures rights change when role changes

  3. Q: CIS CSC differs from NIST CSF in that CIS CSC: - A: Only addresses technical security controls and lacks risk management focus

  4. Q: Which type of security policy addresses employee internet use? - A: Issue-Specific Security Policy (ISSP) — specifically, an Acceptable Use Policy

Section 5.2 — Physical Controls

  1. Q: Which fire suppression system is most appropriate for a computer room? - A: FM-200 or Novec 1230 (clean agents; safe for electronics)

  2. Q: Which physical control prevents piggybacking? - A: Mantrap/airlock

  3. Q: What is the purpose of a UPS in information security? - A: Provides power for seconds to 30 minutes during outages to allow proper shutdown

Section 5.3 — IAM

  1. Q: An employee who has transferred between three departments over 5 years has accumulated access rights from all previous roles. This is BEST described as: - A: Privilege creep

  2. Q: Which biometric error type is MORE dangerous from a security perspective? - A: FAR (Type 2 error) — allows unauthorized users access

  3. Q: Which biometric has the lowest (best) CER? - A: Palm geometry/hand geometry

  4. Q: The BEST defense against rainbow table attacks on passwords is: - A: Salting (adding random data to passwords before hashing)

  5. Q: Which protocol adds an identity layer to OAuth 2.0? - A: OpenID Connect (OIDC)

  6. Q: Which IAM access control model is most commonly used in industry? - A: RBAC (Role-Based Access Control)

Section 5.4 — Network Security

  1. Q: Which type of firewall architecture provides the HIGHEST level of security? - A: Screened subnet (DMZ) — provides three layers of protection

  2. Q: A WAF differs from an NGFW in that: - A: WAF protects web applications at Layer 7; NGFW provides network protection at Layers 3-4

  3. Q: Which VPN protocol is most appropriate for mobile users who frequently switch networks? - A: IKEv2 (supports fast reconnection)

Section 5.5 — DLP

  1. Q: An employee is sending sensitive data via USB drive. Which DLP type would detect this? - A: Endpoint-based DLP

  2. Q: DLP analysis method that uses database records to match against data in transit: - A: Structured fingerprinting (Exact Data Matching - EDM)

Section 5.6 — Encryption

  1. Q: Which encryption type is recommended for bulk data encryption? - A: Symmetric encryption (AES)

  2. Q: Digital signatures provide: - A: Integrity, Authentication, and Nonrepudiation (NOT confidentiality)

  3. Q: What is a digital envelope? - A: A combination of symmetric encryption (for the message) and asymmetric encryption (for the symmetric key)

  4. Q: Kerberos presents a significant risk because: - A: The KDC is a single point of failure

  5. Q: DNSSEC is intended to prevent: - A: DNS cache poisoning and DNS false zones

  6. Q: The primary advantage of OCSP stapling is: - A: Improved performance and privacy (server caches and attaches OCSP response)

Section 5.7 — PKI

  1. Q: A compromised root CA has what effect? - A: Breaks the entire chain of trust; all certificates below it become untrustworthy

  2. Q: Which PKI component verifies certificate revocation in real time? - A: OCSP (Online Certificate Status Protocol)

Section 5.8 — Cloud

  1. Q: In the shared responsibility model, which security aspect is ALWAYS the customer's responsibility? - A: Data security and GRC (Governance, Risk, Compliance)

  2. Q: Which is always the CSP's responsibility? - A: Physical security of the cloud infrastructure

  3. Q: Which cloud deployment model has the highest security and is best for sensitive data? - A: Private cloud

  4. Q: A switched virtual circuit (SVC) is MORE secure than a permanent virtual circuit (PVC) because: - A: It is disassembled after each use, reducing exposure time

  5. Q: Containers differ from VMs in that containers: - A: Share the host OS kernel (not isolated OS per container)

Section 5.9 — Mobile/Wireless/IoT

  1. Q: Which wireless security standard should be avoided? - A: WEP (and LEAP)

  2. Q: WPA3 addresses WPA2 vulnerabilities primarily through: - A: SAE (Simultaneous Authentication of Equals) for key exchange

  3. Q: Remote wiping of a mobile device does NOT guarantee: - A: Permanent data removal (data recovery utilities can recover wiped data)

Section 5.10 — Security Awareness

  1. Q: Which group presents the GREATEST security risk in terms of information security? - A: Operational staff and system users (least technical knowledge)

  2. Q: Security awareness, training, and education differ primarily in: - A: Depth and focus — Awareness (what), Training (how), Education (why)

Section 5.11 — Attacks

  1. Q: Which attack type cannot be defended by antivirus software? - A: Zero-day exploit (unknown vulnerability; no signature available)

  2. Q: A disgruntled developer codes a program to delete files on the date of their expected termination. This is: - A: A logic bomb

  3. Q: Salami attacks are characterized by: - A: Slicing small amounts from many transactions, difficult to detect individually

  4. Q: Ransomware uses what type of encryption? - A: Asymmetric encryption (attacker withholds private key)

Section 5.12 — Security Testing

  1. Q: Management must ALWAYS be informed before: - A: Conducting penetration testing (prior permission required)

  2. Q: White box testing provides access to: - A: Source code, architecture, algorithms (developer's perspective)

  3. Q: Which should be done BEFORE penetration testing? - A: Vulnerability assessment

  4. Q: A blue team is responsible for: - A: Continuous detection and mitigation of cyberattacks

Section 5.13 — Monitoring

  1. Q: An IDS is NOT a substitute for a firewall because: - A: IDS only detects and alerts; it does not block; firewall actively filters traffic

  2. Q: SIEM differs from a log management system in that SIEM: - A: Correlates events from multiple sources in real time and generates alerts

Section 5.14 — Incident Response

  1. Q: When a ransomware infection is confirmed, the FIRST action should be: - A: Quarantine the infected machine and contain spread

  2. Q: An organization should NOT pay a ransomware demand because: - A: It funds criminal activity, and payment does not guarantee decryption

Section 5.15 — Forensics

  1. Q: Why should a computer suspected of being evidence NOT be turned on or rebooted? - A: Booting writes data to the hard drive, potentially destroying evidence and breaking chain of custody

  2. Q: What tool ensures data integrity during forensic investigation? - A: Write blocker (prevents modifications to evidence)

  3. Q: Chain of custody in digital forensics documents: - A: Who had access, procedures followed, and proof that copies are identical to originals

  4. Q: Live analysis in computer forensics is used to: - A: Analyze volatile data (RAM, cache) while the computer is running

🧠 MNEMONICS AND MEMORY TRICKS

CIA Triad

"CIA is in control" - Confidentiality — Only authorized can read - Integrity — Data is accurate and unchanged - Availability — Data is accessible when needed

NIST CSF Five Functions

"I Protect Data, Respond Reliably" - Identify - Protect - Detect - Respond - Recover

Kerberos Components

"KDC Gives Tickets At Service" - KDC = Key Distribution Center - TGS = Ticket-Granting Server - TGT = Ticket-Granting Ticket - AS = Authentication Server - ST = Service Ticket

Biometric Error Types

"FAT - False Acceptance is Type 2" - FAR = Type 2 = False Acceptance (lets bad people IN — dangerous!) - FRR = Type 1 = False Rejection (rejects good users — inconvenient but safer) - Lower CER = Better biometric

Biometric Accuracy Order (Best to Worst CER)

"Palm Has Impressive Retina, Fingerprint Varies" - Palm - Hand geometry - Iris - Retina - Fingerprint - Voice

Access Control Models — MAC vs DAC

  • MAC = Mandatory = Military = Must follow (system enforced, cannot change)
  • DAC = Discretionary = Data owner's Decision (user controlled)
  • RBAC = Role-Based = Really Business-focused (most common)

Firewall Architecture Security Level

"Simple > Dual > Screened" (security increases) - Screened-host — moderate - Dual-homed — higher - Screened-subnet (DMZ) — HIGHEST

Cloud Service Models (Bottom to Top)

"I Put Software On Cloud" - IaaS → Infrastructure (servers, storage) - PaaS → Platform (dev environment) - SaaS → Software (complete apps)

VPN Protocol Security

IPSec Tunnel mode > IPSec Transport mode > L2TP/IPSec > SSTP > IKEv2 > OpenVPN (generally) - Tunnel = Total packet protected (entire IP packet) - Transport = Top layer only (data only)

Ransomware Response — DON'T

  • Don't pay (usually) — funds criminals
  • Don't turn off system — preserves volatile memory
  • Don't ignore quarantine — ransomware spreads

Password Attacks — What Defeats Each

  • Rainbow table → Salt it!
  • Brute force → Lockout after 5 attempts
  • Dictionary → Complexity + history
  • Phishing → Awareness training
  • Credential stuffing → MFA required

Digital Signature provides (NOT):

"IAN — but NOT Cop" (🧑 named IAN) - Integrity ✓ - Authentication ✓ - Nonrepudiation ✓ - Confidentiality ✗ (NOT provided by digital signatures) - Ownership prevention ✗ (cannot prevent copying or theft)

PKI Certificate Revocation Reasons

"ACE SU CA" — The ACE Suspect is an Unspecified CA compromise - Affiliation changes - Compromise of private key - Expiration (cessation of operation) - Superseded - Unspecified - CA compromise

Incident Response Phases (NIST)

"PDCR" — Police Departments Control Responses - Preparation - Detection and Analysis - Containment, Eradication & Recovery - Review (Post-Incident Activity)

Forensic Phases Summary

"FSEC-DATA-DER" - First response - Search and seizure - Evidence collection - Custody (evidence security) - Data acquisition - Analysis - Testing against incident (assessment) - And documentation/reporting - Expert witness testimony (if needed)

Wireless Security Evolution

WEP → WPA → WPA2 → WPA3 (getting stronger) - WEP = Weak, Easily Penetrated - WPA3 = Current Winner, Protects All, with SAE handshake

Symmetric vs. Asymmetric Quick Remember

  • Symmetric = Same key = Same speed = Suited for bulk
  • Asymmetric = Always two keys = Authentication = Always slower
